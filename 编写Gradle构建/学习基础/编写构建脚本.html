## [](#sec:the_script_api)[脚本API](#sec:the_script_api) 当Gradle执行Groovy构建脚本（`.gradle`）时，它将脚本编译为实现[Script]()的类。这意味着`Script`在脚本中可以使用接口声明的所有属性和方法。 当Gradle执行Kotlin构建脚本（`.gradle.kts`）时，它将脚本编译为[KotlinBuildScript](https://gradle.github.io/kotlin-dsl-docs/api/org.gradle.kotlin.dsl/-kotlin-build-script/index.html)的子类。这意味着该`KotlinBuildScript`类型声明的所有可见属性和函数在脚本中都可用。另请分别参见[KotlinSettingsScript](https://gradle.github.io/kotlin-dsl-docs/api/org.gradle.kotlin.dsl/-kotlin-settings-script/index.html)和[KotlinInitScript](https://gradle.github.io/kotlin-dsl-docs/api/org.gradle.kotlin.dsl/-kotlin-init-script/index.html)类型以获取设置脚本和初始化脚本。 ## [](#sec:declaring_variables)[声明变量](#sec:declaring_variables) 可以在构建脚本中声明两种变量：局部变量和额外属性。 ### [](#sec:local_variables)[局部变量](#sec:local_variables) 局部变量用`def`关键字声明。它们仅在声明它们的范围内可见。局部变量是基础Groovy语言的功能。 例子2.使用局部变量 `Groovy``Kotlin` build.gradle def dest = "dest" task copy\(type: Copy\) \{ from "source" into dest \}   1 def dest \= "dest" 2 3 task copy\(type: Copy\) \{ 4 from "source" 5 into dest 6 \} ### [](#sec:extra_properties)[额外的属性](#sec:extra_properties) Gradle的域模型中的所有增强对象都可以容纳额外的用户定义属性。这包括但不限于项目，任务和源集。 可以通过拥有对象的`ext`属性添加，读取和设置其他属性。或者，`ext`可以使用一个块一次添加多个属性。 例子3.使用额外的属性 `Groovy``Kotlin` build.gradle plugins \{ id 'java' \} ext \{ springVersion = "3.1.0.RELEASE" emailNotification = "build\@master.org" \} sourceSets.all \{ ext.purpose = null \} sourceSets \{ main \{ purpose = "production" \} test \{ purpose = "test" \} plugin \{ purpose = "production" \} \} task printProperties \{ doLast \{ println springVersion println emailNotification sourceSets.matching \{ it.purpose == "production" \}.each \{ println it.name \} \} \}   1 plugins \{ 2 id 'java' 3 \} 4 5 ext \{ 6 springVersion \= "3.1.0.RELEASE" 7 emailNotification \= "build\@master.org" 8 \} 9 10 sourceSets.all \{ ext.purpose \= null \} 11 12 sourceSets \{ 13 main \{ 14 purpose \= "production" 15 \} 16 test \{ 17 purpose \= "test" 18 \} 19 plugin \{ 20 purpose \= "production" 21 \} 22 \} 23 24 task printProperties \{ 25 doLast \{ 26 println springVersion 27 println emailNotification 28 sourceSets.matching \{ it.purpose \== "production" \}.each \{ println it.name \} 29 \} 30 \} 输出**`gradle -q printProperties`** \> gradle -q printProperties 3.1.0。发布 build\@master.org 主要 插入 在此示例中，`ext`块将两个额外的属性添加到`project`对象。此外，`purpose`通过将设置`ext.purpose`为`null`（`null`是允许的值），将命名属性添加到每个源集。添加属性后，就可以像预定义属性一样读取和设置它们。 通过要求添加属性的特殊语法，当尝试设置（预定义或额外的）属性但该属性拼写错误或不存在时，Gradle可能会快速失败。可以从可以访问其所属对象的任何位置访问其他属性，从而使它们的范围比局部变量更大。从子项目中可以看到项目的其他属性。 有关其他属性及其API的更多详细信息，请参阅API文档中的[ExtraPropertiesExtension]()类。 ##
